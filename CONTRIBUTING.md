# Contributing

## Purpose

This repository is maintained as a strategic portfolio project and practical blueprint for enterprise digitalization and AI readiness.

Contributions should improve one of these outcomes:

- clarity of the transformation framework
- applicability for real companies
- quality of templates and assessment logic
- usefulness for leadership, IT, architecture, or program delivery

## Contribution Principles

- keep documents decision-oriented
- prefer practical structure over theory-heavy writing
- make assumptions explicit
- keep language clear and executive-readable
- connect recommendations to measurable outcomes

## Suggested Contribution Areas

- additional industry-specific case studies
- sector-specific regulatory considerations
- stronger KPI and governance examples
- implementation playbooks by domain
- improved diagrams and visuals

## Content Standards

- use concise headings
- prefer ASCII text where possible
- avoid buzzword-heavy language without operational meaning
- explain tradeoffs, not just recommendations
- keep reusable templates generic enough for multiple organizations

## Pull Request Guidance

1. Explain what problem the change solves.
2. State which audience benefits from the change.
3. Call out any assumptions or limitations.
4. Keep edits scoped and coherent.

## Review Focus

Changes should be reviewed for:

- strategic consistency
- applicability in German enterprise environments
- governance realism
- business relevance
- readability for public GitHub audiences
